Quick start

Deploy your first project to Concreto in under five minutes. Connect a Git repository and let Kubernetes handle the rest — no infrastructure knowledge required.

On this page
  • Prerequisites
  • Deploy via dashboard
  • Deploy via CLI
  • Your project is live
  • What's next

Prerequisites

Before you begin, make sure you have:

  • A Concreto account — sign up for free, no credit card needed.
  • A project hosted in a Git repository (GitHub, GitLab, or any public URL). Concreto supports Node.js, Python, Go, and static sites out of the box.
  • Optionally, the Concreto CLI installed for command-line deploys.

Deploy via the dashboard

The easiest way to get started is through the web dashboard — no terminal required.

  1. 1
    Sign in and open the dashboard

    Go to your Concreto dashboard and click + New application in the top right.

  2. 2
    Enter your Git repository URL

    Give your application a name and paste the Git repository URL (e.g. https://github.com/you/my-app). Concreto will clone and analyse the project automatically.

  3. 3
    Click Deploy

    ArgoCD picks up the repository, runs the build pipeline, and provisions a Kubernetes pod. Watch the log output in real time from the Deployments view.

  4. 4
    Your app is live

    Once the status turns Running, your project is accessible at a *.concreto.run subdomain. CPU, memory, and request metrics are available under the Metrics tab.

Deploy via the CLI

For faster iteration and CI/CD integration, use the Concreto CLI.

Install the CLI

bash 
# Install via npm
npm install -g @concreto/cli

# Or via Homebrew (macOS / Linux)
brew install concreto

# Verify installation
concreto --version

Authenticate

bash 
concreto login
# Opens your browser to complete authentication
✔ Logged in as you@company.com

Deploy your project

bash 
# Deploy from a Git repository URL
concreto deploy --repo https://github.com/you/my-app

# Or deploy from the current directory (reads .git remote)
concreto deploy

The CLI reads a concreto.json config file if present in your project root. This lets you commit your deploy settings alongside your code.

Example concreto.json

json 
{
  "name": "my-api",
  "repository": "https://github.com/you/my-api",
  "runtime": "node20",
  "build": "npm run build",
  "start": "npm start",
  "port": 3000
}
ℹ
The port field is optional. Concreto automatically exposes port 3000 for Node.js, 8000 for Python, and 8080 for Go unless overridden. All traffic is routed through the Kubernetes Ingress controller over HTTPS.

Your project is live

After a successful deploy, you'll receive a URL like:

output 
✔ Deployment live in 21s
→ my-api.concreto.run
  SHA: a1b2c3d  ·  Cluster: eu-west  ·  Runtime: Node 20 LTS

Your app is now publicly accessible with a free HTTPS certificate provisioned automatically via the Kubernetes Ingress controller. Monitor CPU, memory, and request rates from the Metrics tab in your dashboard.

⚡
Free tier deployments are limited to 3 applications and 50 deployments per month. Upgrade to Pro for unlimited applications and 500 deployments/month.
